## 一、产品定位与核心哲学

### 1.1 产品定位

> **TimeTracker 不是计时器，不是统计工具，是你人生时间的镜子。**

大多数时间 App 解决的是"记录"问题。TimeTracker 解决的是"认知"问题。

| 对比维度 | 普通时间 App | TimeTracker |
|----------|-------------|-------------|
| 核心价值 | 记录花了多少时间 | 认知时间结构是否合理 |
| 数据呈现 | 饼图 + 柱状图 | 结构分析 + 趋势变化 + 质量评分 |
| 用户感受 | 数据罗列 | 认知冲击 → 情绪触发 → 行为改变 |
| 差异化 | 无 | 基于柳比歇夫方法论，输出占比为核心 |

### 1.2 柳比歇夫方法论精髓

柳比歇夫坚持 56 年时间统计，核心不是"记录时间花在哪里"，而是：

1. **结构（Structure）** — 时间分配是否均衡，产出占比是否合理
2. **趋势（Trend）** — 时间结构是否在变好 / 变差
3. **价值（Value）** — 时间是否用在有长期产出的事情上

### 1.3 产品核心壁垒

1. **可视化时间块（日/周）** → 直观认知，普通 App 难以做到的沉浸感
2. **结构化分类评分（输入/输出/消耗/维持）** → 可量化"产出性"
3. **周复盘与趋势分析** → 让用户有纵向方向感，不只是单日数据
4. **行为引导 + 补缺提示** → 用户持续回流，形成时间记录习惯

## 二、时间分类体系

### 2.1 四大结构分类（核心不可删减）

| 分类 | 含义 | 典型活动 | `isProductive` | 颜色 |
|------|------|----------|----------------|------|
| 输入（Input）| 知识吸收 | 学习 / 阅读 / 听播客 | ✅ | 蓝 |
| 输出（Output）| 知识产出 | 写作 / 编程 / 创作 | ✅ | 绿 |
| 消耗（Consumption）| 娱乐消耗 | 刷短视频 / 游戏 / 无意义社交 | ❌ | 橙/红 |
| 维持（Maintenance）| 生命维持 | 睡眠 / 饮食 / 运动 / 杂务 | ❌（中性）| 紫 |

### 2.2 子分类体系（可扩展）

```
输入
├── 专业学习
├── 通识阅读
├── 在线课程
└── 播客 / 音频

输出
├── 写作
├── 编程 / 开发
├── 创意设计
└── 表达 / 演讲

消耗
├── 短视频 / 娱乐
├── 游戏
├── 社交媒体浏览
└── 无效社交

维持
├── 睡眠
├── 饮食
├── 运动
├── 通勤
└── 家务
```

## 三、数据展示体系（四层递进）

> **设计原则：每屏只讲一个结论，每个图表必须有一句人话总结，让用户产生情绪（焦虑 / 成就 / 反思）。**

### 第一层：时间分布（基础）

**目标感受**：今天/本周时间花在哪里了  
**核心展示**：各分类占比 + 人话总结  
**关键设计**：不只是饼图，要有一句判断性文字

```
示例：
📊 今日时间分布
娱乐占比 38%  ⚠️  已超过建议阈值（< 25%）
```

### 第二层：时间结构（核心差异点）

**目标感受**：我的时间"结构"是否合理  
**核心展示**：输入 / 输出 / 消耗 / 维持 四块比例  
**关键设计**：让用户第一次意识到"我几乎没有输出" → 认知冲击

```
示例：
🧱 今日时间结构
输入   25%  ████████░░░░
输出    8%  ███░░░░░░░░░ ⚠️ 严重不足
消耗   40%  █████████████ ⚠️ 偏高
维持   27%  █████████░░░
```

### 第三层：趋势变化（真正有价值）

**目标感受**：我的时间结构在变好还是变差  
**核心展示**：7天 / 30天每类趋势  
**关键设计**：展示"变化"而非"快照"

```
示例：
📈 输出时间趋势
本周：6%
上周：12%  ↓ 下降了 50%  ⚠️
```

### 第四层：时间质量评分（杀手功能）

**目标感受**：今天/本周综合时间质量如何  
**核心展示**：综合评分（0~100）+ 扣分原因  
**关键设计**：让用户上瘾，每天想刷新评分

```
评分规则（可配置）：
输出占比 > 15%           +30分
输入占比 > 20%           +20分
娱乐占比 < 25%           +20分
深度工作时段 >= 2         +20分
连续记录天数 bonus        +10分

示例：
今日时间质量：68分 ⚠️
- 输出不足（-30）
- 娱乐偏高（-20）
```

## 四、核心功能需求全景

### 4.1 记录功能（核心入口）

#### F-01 手动开始/结束计时
- 选择分类 → 开始 → 停止（可添加备注）
- 停止时展示本次时长
- 支持修改分类（如果记错了）

#### F-02 快捷按钮记录
- 首页 4 个分类快捷启动卡片
- 有活跃计时时禁用，防止误操作
- 视觉上强调当前活跃计时的分类色

#### F-03 补记录
- 检测未记录时间段（gap > 30 分钟提示）
- 日报页面展示空白块，一键填充
- 日历视图点击空白区直接补记录
- 支持预设时间段自动填入开始/结束时间

#### F-04 记录编辑与删除
- 支持修改分类、时间段、备注、标签
- 删除需确认（不可恢复）

#### F-05 App Intents / Siri 快速记录
- "用 TimeTracker 开始输出计时"
- "用 TimeTracker 停止计时"
- Shortcuts 支持自动化工作流（如：打开 Notion 时自动开始"输出"计时）

---

### 4.2 日历时间块视图（视觉核心）

#### F-06 日视图
- 纵向时间轴（06:00 ~ 24:00，可配置）
- 按真实时间线性排布彩色时间块
- 时间块内展示：分类名 / 起止时间 / 时长
- 空白区域点击 → 快速补记录弹窗
- 时间块点击 → 详情 / 编辑

#### F-07 周视图
- 横向 7 列 × 纵向时间，紧凑模式
- 每格显示分类色块 + 时长
- 点击某天展开日视图
- 顶部显示当周总效率评分

#### F-08 月视图（规划中）
- 日历格式，每格用色块堆叠展示各分类分布
- 适合长周期回顾

#### F-09 时间块拖动编辑（规划中）
- 长按时间块 → 可上下拖动调整时间段
- 直接在视图内修改，不需要进入表单

---

### 4.3 日报（每日分析）

#### F-10 日报总览
- 已记录总时间 / 空白时间 / 效率分（三列）
- 日期前后导航

#### F-11 分类统计
- 各分类：时长 + 占比 + 渐变进度条
- 人话判断（如：消耗偏高、输出严重不足）

#### F-12 空白时段提示
- 列出 30 分钟以上的未记录区间
- 每段旁有"补记录"快速按钮

#### F-13 今日洞察
- 规则引擎生成文本提示（可替换为 AI）
- 典型提示：
  - "今日高效时间不足 30%，建议减少娱乐"
  - "消耗类时间连续 3 天超过 3 小时"
  - "本次深度工作持续 90 分钟，优秀！"

#### F-14 今日记录列表
- 按时间排序，可点击进入详情

---

### 4.4 周报（趋势复盘）

#### F-15 周总览
- 本周总记录时间 / 平均效率 / 记录天数 / 总质量评分

#### F-16 每日效率趋势（柱状图）
- 7 日效率评分柱状图
- 今天高亮，分数颜色（绿/橙/红）

#### F-17 分类占比环形图
- 本周四大分类时间占比
- 与上周对比（↑ ↓ 箭头）

#### F-18 时间雷达图（规划中）
- 4 个轴：输入 / 输出 / 消耗 / 维持
- 理想形态 vs 实际形态
- 让用户直觉感受"我的时间是否失衡"

#### F-19 每日堆叠条形图
- 7 行 × 堆叠颜色，瞬间看出结构差异

#### F-20 周洞察
- "本周输出下降 50%，是什么影响了你？"
- "本周消耗超过 15 小时，注意调整"

## 五、设计原则（非功能性需求）

### 5.1 UI 原则

| 原则 | 说明 |
|------|------|
| 极简操作 | 核心功能 3 步以内完成（快捷启动 1 步）|
| 一屏一结论 | 每个页面只讲一个最重要的信息 |
| 数据就要有结论 | 每个图表或数字旁必须有人话判断文字 |
| 情绪设计 | 让用户产生焦虑（超标）/ 成就（达标）/ 反思（下降）|
| 颜色语义 | 绿色=好 / 橙色=注意 / 红色=警告，跨全 App 一致 |

### 5.2 数据原则

| 原则 | 说明 |
|------|------|
| 实时计算 | 汇总数据（日/周）实时由 TimeEntry 计算，不存储冗余 |
| 可补记录 | 任何时候都能填补过去的时间空白 |
| 绝对值 + 占比 | 同时展示小时数和百分比，避免相对数失真 |
| 长期累积 | 提供累积账本视图，增强长期价值感 |

### 5.3 习惯养成原则

- 补记录零门槛：空白时间段一键选择分类即可
- 评分上瘾机制：每日评分让用户有"刷新动力"
- 连续记录奖励：连续记录天数在 UI 中可见（类似 Duolingo streak）
- 晚间提醒：温和提示补记录，不强制打扰

## 八、柳比歇夫"效率金字塔"目标参考

> 以下为理想的时间结构参考区间，可作为评分基准

| 分类 | 建议占比 | 说明 |
|------|----------|------|
| 输出 | 15~30% | 核心产出，低于此值需警示 |
| 输入 | 20~30% | 知识积累，支撑输出的原料 |
| 维持 | 30~40% | 生命运作必要成本 |
| 消耗 | < 20% | 娱乐/恢复，超标则拖累整体 |

## 九、成功指标

| 指标 | 定义 | 目标 |
|------|------|------|
| 日活连续性 | 连续7天以上记录的用户比例 | > 40% |
| 补记录率 | 空白时间 < 4h/天 的用户比例 | > 60% |
| 周报查看率 | 打开周报的活跃用户比例 | > 70% |
| 输出分类增长 | 使用3个月后用户输出类时间占比 | 平均提升 5% |
