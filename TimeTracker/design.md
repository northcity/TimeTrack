# TimeTracker 全新 UI/UX 设计规范与风格总结 (2026版)

## 1. 核心设计理念 (Core Philosophy)
本次重构彻底抛弃了早期较为初级的“列表式”呈现，转向**“现代化、沉浸式、数据可视化”**的产品设计路线。整体风格融合了市面上顶级的习惯追踪应用（如 Structured、Tide、Timetrack）以及 Apple 系统级应用（Apple Calendar）的美学。

重点落在三处：**清晰的功能视觉层级、直观的时间块感知、无缝的交互体验**。

## 2. 首页计时组件 (TimerView) - “沉浸式与极简反馈”
- **视觉重心聚焦**：采用超大醒目的倒计时/正计时表盘作为绝对的视觉中心，利用高斯模糊、半透明材质和动态进度环（Progress Ring）来提供呼吸感，让用户进入“心流”状态。
- **现代化卡片布局**：分类（Category）与子分类（SubCategory）的选取抛弃了繁琐的表单，改为横向滑动的优雅胶囊按钮（Capsule）配合强调色，点击即选，即刻开始。
- **状态即反馈**：一旦开始追踪，背景或控制面板会随之进入“专注模式”样式，停止按钮使用警示但柔和的色调。

## 3. 日历与时间轴引擎 (CalendarBlockView) - “原生级精密排版”
彻底重写了日历模块，打造了一个支持**“日、周、月、年”**四大维度交叉下钻的专业级时间可视化面板。

- **日视图 (Day Timeline)**：
  - **仿 Apple Calendar 布局**：左侧提供 0-24 小时绝对时间轴，右侧渲染时间块。
  - **动态碰撞算法 (TimelineLayoutEngine)**：自研图渲染算法（基于区间图着色），完美解决同一时间段内多个任务的**穿插、重叠**问题。时间块不再拥挤遮挡，而是自适应平分屏幕宽度。
  - **当前时间指示器**：红色横线与小圆点实时指示当前时间，极具系统原生感。

- **周视图 (Week Timeline)**：
  - 以紧凑的七列视图并排展示本周 7 天的 24 小时时间轴，使得一周的时间分配密度与节奏一目了然。

- **月视图 (Month Calendar)**：
  - 采用经典的 7x5 网格布局，通过在日期下方的“微型色彩圆点（Categories Dots）”展示当天的主要活动构成，简洁而不失信息量。
  
- **年视图 (Year Heatmap)**：
  - 引入了类似 GitHub Commit / Apple Fitness 的**“年度热力图 (Heatmap)”**。通过 12 个月的微型网格块和深浅不同的蓝色，直观反映使用者全年的活跃度和投入热度。

## 4. 色彩与排版体系 (Color & Typography)
- **卡片化与细边框**：大量采用 `RoundedRectangle`（圆角 3-4pt）、低透明度填充（`opacity(0.2)`）外加同色系稍深的细边框（`strokeBorder`）组合。即保证了各自分类的特征色，又能使画面保持清爽。
- **排版 (Typography)**：摒弃了默认的大字号，辅助信息（如持续时间、子标签）大量采用 `.caption` / `.caption2` 配合 `.secondary` 颜色，腾出视觉空间给最核心的数据。

## 5. 总结
本次改版后，TimeTracker 的 UI 标准已经与成熟商业级工具看齐。它不仅在“计时记录”动作上更加顺滑干脆，更通过先进的时间轴算法，让“数据回顾”阶段具有了极强的可读性与专业感。
