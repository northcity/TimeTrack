## 一、产品定位与核心哲学

### 1.1 产品定位

> **一句话定位：帮助知识工作者看清时间去向，提升产出占比。**

#### 我们解决的核心问题

用户的真实痛点不是"不知道时间花在哪"，而是：

- 🔴 **结构盲区**：感觉很忙，但说不清时间花在了有价值的事情上还是被消耗掉了
- 🔴 **趋势失察**：不知道自己的时间分配是在变好还是变差
- 🔴 **行为惯性**：即使知道问题，也缺乏持续改变的动力和反馈

TimeTracker 的价值主张：**让时间结构可视化、可量化、可追踪，用数据驱动行为改变。**

#### 产品体验公式

```
认知冲击（看清真相）→ 情绪触发（焦虑/成就/后悔）→ 行为改变（调整分配）→ 正反馈（评分提升）→ 上瘾循环
```

> **关键洞察**：图表和数据是手段，不是目的。
> 真正驱动用户持续使用的是**每天一句扎心的人话**，而不是一张精美的饼图。

#### 目标用户画像

| 用户类型 | 核心诉求 | 使用场景 |
|----------|----------|----------|
| 知识工作者 | 提升深度工作时间占比 | 日常工作复盘、周末反思 |
| 学生/考研党 | 确保学习时间足够、减少娱乐消耗 | 备考期间自律监控 |
| 内容创作者 | 平衡输入与输出、追踪创作投入 | 内容产出周期管理 |
| 自我提升者 | 长期习惯养成、可视化成长轨迹 | 年度复盘、人生规划 |

**非目标用户**：需要精确计费的自由职业者（推荐 Toggl）、团队协作时间管理（推荐 Clockify）

#### 与同类产品的差异

| 维度 | 传统时间追踪（Toggl/RescueTime） | TimeTracker |
|------|----------------------------------|-------------|
| 核心视角 | 时间花在哪个项目/应用 | 时间结构是否健康（输入/输出/消耗/维持） |
| 数据呈现 | 项目维度的统计报表 | 结构分析 + 趋势变化 + 质量评分 |
| 设计目标 | 记录与统计 | 认知冲击 → 情绪触发 → 行为改变 |
| 理论基础 | 无特定方法论 | 柳比歇夫时间统计法 |

### 1.2 柳比歇夫方法论精髓

柳比歇夫坚持 56 年时间统计，他的方法论核心是一个递进的认知链条：

```
记录 → 结构化 → 可视化 → 趋势追踪 → 行为调整
```

三个核心维度：

1. **结构（Structure）** — 时间分配是否均衡，产出占比是否合理
   - 不只是"今天学了多久"，而是"学习在全天时间中占多少"
   
2. **趋势（Trend）** — 时间结构是否在变好 / 变差
   - 单日数据没有意义，周/月趋势才能反映真实状态
   
3. **价值（Value）** — 时间是否用在有长期产出的事情上
   - 区分"输入/输出"vs"消耗/维持"，量化产出性

**为什么这套方法有效？**

- 人类对抽象时间没有直觉，可视化结构能产生认知冲击
- 连续记录形成数据积累，趋势比快照更有说服力
- 评分机制提供即时反馈，满足成就感需求

### 1.3 产品核心壁垒

#### 功能层壁垒（易被复制，但形成体验差异）

1. **可视化时间块（日/周）** → 直观认知，沉浸式时间感知
2. **结构化分类评分（输入/输出/消耗/维持）** → 可量化"产出性"
3. **周复盘与趋势分析** → 让用户有纵向方向感
4. **行为引导 + 补缺提示** → 降低记录门槛

#### 深层壁垒（真正的护城河）

| 壁垒类型 | 说明 |
|----------|------|
| **数据积累** | 用户记录越久，历史数据越有价值，迁移成本越高 |
| **习惯养成** | 连续记录 streak 机制 + 评分上瘾 → 形成每日打开习惯 |
| **认知绑定** | 用户一旦接受"输入/输出/消耗/维持"框架，会用这套语言思考时间 |
| **方法论背书** | 柳比歇夫 56 年实践验证，赋予产品权威感和故事性 |

### 1.4 上瘾机制设计（情绪驱动留存）

> 产品逻辑再好，用户不打开就是零。以下机制解决"为什么每天都想打开"的问题。

#### 机制一：每日人话评价（核心情绪触发点）

比任何图表都重要的一句话，放在日报最顶部：

```
犀利风格（默认）：
「今天 80% 的时间在输入，但几乎没有产出——你在逃避创作吗？」
「连续 3 天消耗超过 4 小时，你的时间正在被吞噬」
「今天是高质量一天，超过你过去 90% 的日子」

温和风格（可切换）：
「今天输出时间不多，明天试试先做最难的事？」
「消耗有点多，但没关系，意识到就是改变的开始」
```

**设计要点**：
- 文案必须像「一个了解你的朋友说的话」，不是系统通知
- 基于当日数据 + 历史趋势生成，有对比才有冲击力
- Phase 1-3 用规则引擎模板，Phase 4 接入 LLM 个性化生成

#### 机制二：后悔换算（时间的机会成本）

将消耗时间转换为用户能感知的具体事物：

```
「本周你花了 11 小时刷短视频 = 读完 1.5 本书 / 写 3 篇文章 / 学完 1 门课的 60%」
「过去一个月你在娱乐上花了 48 小时 = 一个完整的工作周」
「如果把这些时间用于输出，你可以多写 15,000 字」
```

**设计要点**：
- 换算基准可配置（1本书 ≈ 7小时，1篇文章 ≈ 3小时，1门课 ≈ 20小时）
- 在日报和周报的消耗统计旁展示
- 不是说教，是客观事实呈现——让用户自己产生 FOMO

#### 机制三：与过去的自己对比

没有服务端，无法做跨用户对比，但可以做更有意义的事——跟自己的历史比：

```
「本周输出占比 18%，比上周提升了 6 个百分点 ↑ 你在变好」
「这是你连续第 12 天记录，历史最长纪录是 23 天——快破纪录了」
「本月消耗时间比上月减少 8 小时，节省的时间去了输入 +5h 和输出 +3h」
```

#### 机制四：成就徽章体系

| 徽章 | 触发条件 | 情绪 |
|------|----------|------|
| 🔥 深度工作者 | 单次专注 >= 90 分钟 | 成就感 |
| 📚 知识海绵 | 单日输入 >= 3 小时 | 鼓励 |
| ✍️ 高产创作者 | 连续 3 天输出 > 输入 | 骄傲 |
| 🎯 结构大师 | 周评分 >= 85 | 自豪 |
| ⚡ 连续记录 | 7 / 30 / 100 天不断 | 坚持 |
| 😱 时间黑洞 | 单日消耗 > 6 小时 | 警醒 |

### 1.5 分类认知引导（解决用户犹豫问题）

> **核心原则**：不替用户判断，但给用户足够的决策辅助，让分类在 2 秒内完成。

用户在"这算输入还是消耗？"上犹豫超过 5 秒 = 放弃记录的开始。

#### 判断决策树（App 内展示为引导卡片）

| 问自己 | 是 → | 否 → |
|--------|------|------|
| 这段时间有没有产出具体成果？ | **输出** | ↓ |
| 这段时间有没有学到新东西？ | **输入** | ↓ |
| 这段时间是生活必需吗？ | **维持** | **消耗** |

#### 常见争议场景预设答案

| 场景 | 建议分类 | 理由 |
|------|----------|------|
| 刷技术视频 | 输入 | 有学习意图且有收获 |
| 无目的刷短视频 | 消耗 | 无明确学习目标 |
| 和朋友深度交流 | 输入/输出 | 有信息交换和思考 |
| 闲聊八卦 | 消耗 | 无实质信息增量 |
| 运动健身 | 维持 | 身体维护，非直接产出 |
| 通勤听播客 | 输入 | 利用碎片时间学习 |
| 做饭（认真研究新菜） | 输入 | 有技能提升 |
| 做饭（日常） | 维持 | 日常生活维护 |

#### 智能默认（降低首次使用门槛）

- 新用户首次使用时展示分类引导卡片（上述决策树）
- 子分类有默认映射，用户选子分类后自动归入主分类
- 记录历史学习：如果用户多次将同一备注归为某分类，下次自动推荐

### 1.6 增长与传播设计

> 当前阶段为本地 App，无服务端。增长策略以"内容传播"为主，不依赖社交功能。

#### 已实现

- ✅ 周报分享卡片（F-36）：可分享到社交媒体
- ✅ CSV 数据导出（F-35）：满足数据控需求

#### 待实现（高 ROI）

## 二、时间分类体系

### 2.1 四大结构分类（核心不可删减）

| 分类 | 含义 | 典型活动 | `isProductive` | 颜色 |
|------|------|----------|----------------|------|
| 输入（Input）| 知识吸收 | 学习 / 阅读 / 听播客 | ✅ | 蓝 |
| 输出（Output）| 知识产出 | 写作 / 编程 / 创作 | ✅ | 绿 |
| 消耗（Consumption）| 娱乐消耗 | 刷短视频 / 游戏 / 无意义社交 | ❌ | 橙/红 |
| 维持（Maintenance）| 生命维持 | 睡眠 / 饮食 / 运动 / 杂务 | ❌（中性）| 紫 |

```
输出

├── 创意设计

消耗
├── 短视频 / 娱乐

维持
├── 睡眠

```

## 三、数据展示体系（四层递进）

> **设计原则：每屏只讲一个结论，每个图表必须有一句人话总结，让用户产生情绪（焦虑 / 成就 / 反思）。**

### 第一层：时间分布（基础）

**目标感受**：今天/本周时间花在哪里了  
**核心展示**：各分类占比 + 人话总结  
**关键设计**：不只是饼图，要有一句判断性文字

```
示例：
📊 今日时间分布
娱乐占比 38%  ⚠️  已超过建议阈值（< 25%）
```

### 第二层：时间结构（核心差异点）

**目标感受**：我的时间"结构"是否合理  
**核心展示**：输入 / 输出 / 消耗 / 维持 四块比例  
**关键设计**：让用户第一次意识到"我几乎没有输出" → 认知冲击

```
示例：
🧱 今日时间结构
输入   25%  ████████░░░░
输出    8%  ███░░░░░░░░░ ⚠️ 严重不足
消耗   40%  █████████████ ⚠️ 偏高
维持   27%  █████████░░░
```

### 第三层：趋势变化（真正有价值）

**目标感受**：我的时间结构在变好还是变差  
**核心展示**：7天 / 30天每类趋势  
**关键设计**：展示"变化"而非"快照"

```
示例：
📈 输出时间趋势
本周：6%
上周：12%  ↓ 下降了 50%  ⚠️
```

### 第四层：时间质量评分（杀手功能）

**目标感受**：今天/本周综合时间质量如何  
**核心展示**：综合评分（0~100）+ 扣分原因  
**关键设计**：让用户上瘾，每天想刷新评分

```
评分规则（可配置）：
输出占比 > 15%           +30分
输入占比 > 20%           +20分
娱乐占比 < 25%           +20分
深度工作时段 >= 2         +20分
连续记录天数 bonus        +10分

示例：
今日时间质量：68分 ⚠️
- 输出不足（-30）
- 娱乐偏高（-20）
```

## 四、核心功能需求全景

### 4.1 记录功能（核心入口）

#### F-01 手动开始/结束计时
- 选择分类 → 开始 → 停止（可添加备注）
- 停止时展示本次时长
- 支持修改分类（如果记错了）

#### F-02 快捷按钮记录
- 首页 4 个分类快捷启动卡片
- 有活跃计时时禁用，防止误操作
- 视觉上强调当前活跃计时的分类色

#### F-03 补记录
- 检测未记录时间段（gap > 30 分钟提示）
- 日报页面展示空白块，一键填充
- 日历视图点击空白区直接补记录
- 支持预设时间段自动填入开始/结束时间

#### F-04 记录编辑与删除
- 支持修改分类、时间段、备注、标签
- 删除需确认（不可恢复）

#### F-05 App Intents / Siri 快速记录
- "用 TimeTracker 开始输出计时"
- "用 TimeTracker 停止计时"
- Shortcuts 支持自动化工作流（如：打开 Notion 时自动开始"输出"计时）

- 每格显示分类色块 + 时长

#### F-08 月视图（规划中）
- 日历格式，每格用色块堆叠展示各分类分布

## 五、设计原则（非功能性需求）

### 5.1 UI 原则

| 原则 | 说明 |
|------|------|
| 极简操作 | 核心功能 3 步以内完成（快捷启动 1 步）|
| 一屏一结论 | 每个页面只讲一个最重要的信息 |
| 数据就要有结论 | 每个图表或数字旁必须有人话判断文字 |
| 情绪设计 | 让用户产生焦虑（超标）/ 成就（达标）/ 反思（下降）|
| 颜色语义 | 绿色=好 / 橙色=注意 / 红色=警告，跨全 App 一致 |

### 5.2 数据原则

| 原则 | 说明 |
|------|------|
| 实时计算 | 汇总数据（日/周）实时由 TimeEntry 计算，不存储冗余 |
| 可补记录 | 任何时候都能填补过去的时间空白 |
| 绝对值 + 占比 | 同时展示小时数和百分比，避免相对数失真 |
| 长期累积 | 提供累积账本视图，增强长期价值感 |

### 5.3 习惯养成原则

- 补记录零门槛：空白时间段一键选择分类即可
- 评分上瘾机制：每日评分让用户有"刷新动力"
- 连续记录奖励：连续记录天数在 UI 中可见（类似 Duolingo streak）
- 晚间提醒：温和提示补记录，不强制打扰
